# standup — 現状整理と次タスク確認

作業の再開・区切りで、現状を素早く把握して次の一手を決めるためのスキル。
「やったこと／次やること」を確認するデイリースタンドアップ風のチェックポイント。

## 手順

1. **ROADMAP.md を読む**（リポ直下 `ROADMAP.md`、git 管理外のローカル進捗メモ）。
   完了 / 進行中 / Backlog / 制約 / 環境メモを把握する。
   無ければ `git log` と CLAUDE.md / README.md から再構築して ROADMAP.md を作り直す。

2. **git の状態を確認**して ROADMAP と突き合わせる。
   - `git log --oneline -8` / `git status -sb` / 現在のブランチ
   - マージ済みなのに ROADMAP の「完了」へ未反映の項目があれば拾う
   - 未コミット・未 push の作業が無いか確認する

3. **整理して提示**する（簡潔に）。
   - やったこと（直近の完了）
   - 進行中（あれば。中断箇所も明示）
   - 次の候補：Backlog から依存関係・価値順で 2〜4 個

4. **次にやることをユーザーに確認**する（AskUserQuestion で候補提示）。
   決まったら、必要に応じてブランチを切って着手する。

5. **ROADMAP.md を更新**する。古い／実態とズレていれば、完了の反映・新課題の追加を
   この場で行う（区切りごとに最新化するのが本スキルの役目）。

## 注意

- 設計上の制約を踏まえて次タスクを提案する（依存ゼロ / ReplayKit は遡れない /
  Slack は動画不可 / AI は動画を直接解析不可）。回避できない案は出さない。
- 実機でしか検証できない項目（ReplayKit 実キャプチャ等）は明示する。
- プッシュ等の公開・不可逆操作は実行前に必ず確認を取る。
